## О проекте

**Foodify** — это мобильное приложение, которое помогает быстрее и удобнее анализировать продукты в повседневной жизни.

Изначально проект создавался как личный инструмент для ответа на простые, но важные вопросы при покупке:

- какие у продукта **КБЖУ**;
- есть ли в составе **Е-добавки**;
- насколько у него **хороший состав**;
- как он выглядит на фоне **альтернативных товаров**.

Со временем идея расширилась, и приложение стало объединять не только анализ продуктов, но и сопутствующие сценарии:

- **карты лояльности**;
- **список покупок**;
- **сравнение товаров**;
- **калькулятор КБЖУ**.

Отдельную роль в проекте играет Роскачество. При наличии данных по товару Роскачество становится одним из наиболее полезных и достоверных источников. Оно уже используется в приложении и может лечь в основу дальнейшего расширения функционала.

---

## Зачем нужен Foodify

Покупка продуктов — это постоянная бытовая рутина, в которой обычно не хватает удобного инструмента “в одном месте”.

**Foodify** решает эту задачу, объединяя:

- сканирование товаров по штрих-коду;
- анализ состава;
- расшифровку добавок;
- сравнение продуктов;
- инструменты для покупок и хранения скидочных карт.

То есть это не просто “сканер штрих-кодов”, а попытка собрать в одном приложении все, что помогает быстрее принять решение о покупке (по крайней мере, чтобы мне было удобнее покупать)

## Основные возможности

### Анализ продуктов
- Сканирование товара по **штрих-коду**
- Поиск карточки продукта из нескольких источников
- Просмотр **КБЖУ**
- Анализ состава, включая метрики Nutri-Score, Nova и "Светофор" (российский аналог Nutri-Score)
- Поиск и описание **Е-добавок**

### Сравнение
- Сравнение до **3 продуктов**
- Сопоставление по ключевым метрикам
- Процентное соотношение по выбранным показателям
- Основа для будущего сравнения по отзывам

### Повседневные инструменты
- **Список покупок**
- **Карты лояльности**
- **Калькулятор КБЖУ**
- Напоминания для покупок
- Голосовой ввод

## Как работает поиск продукта

Самая сложная часть проекта — это поиск продукта по штрих-коду.

### Основная проблема
В России нет единой открытой базы товаров, где одновременно были бы доступны:

- корректное наименование;
- состав;
- КБЖУ;
- характеристики;
- данные о качестве;
- дополнительные сведения по продукту.

Из-за этого поиск построен как **цепочка источников**, где каждый следующий используется только в том случае, если предыдущий не смог вернуть данные.

## Архитектура поиска

<img width="500" height="700" alt="примерная архитектура поиска по ШК" src="https://github.com/user-attachments/assets/fda56544-5505-4b78-98f4-33c86fa6a35e" />

В основе подхода — паттерн **Chain of Responsibility**.

### Последовательность поиска

#### 1. Получение штрих-кода
Через **Zxing.MAUI** приложение сканирует штрих-код товара.

Сейчас основной сценарий — это **EAN-13**, а в будущем планируется поддержка **DataMatrix**.
#### 2. Роскачество
Приоритетный источник — **официальное API Роскачества**.

Если товар найден, можно получить:
- подробные обзоры;
- характеристики;
- результаты исследований;
- сведения о качестве продукта.

#### 3. Национальный каталог
Если данных в Роскачестве нет, приложение обращается к **Национальному каталогу**.

По ощущениям, этот сервис должен выступать агрегатором продуктовых товаров, так как официальное приложение **Честного знака** ведет именно туда.

Плюсы:
- хорошие карточки товаров;
- детальные характеристики;
- особенно полезные данные по молочной продукции и воде.

Минусы:
- заметная часть товаров, вероятно, скрыта или недоступна.

#### 4. OpenFoodFacts
Следующий этап — **OpenFoodFacts**.

Плюсы:
- удобное открытое API;
- понятная структура данных.

Минусы:
- база пополняется пользователями;
- карточки российских товаров часто заполнены неполно;
- иногда информация отсутствует вовсе.

#### 5. Поиск названия и попытка получить карточку через Яндекс Еду
Если товар все еще не найден, приложение пытается:

1. определить название товара через открытые базы штрих-кодов;
2. затем по названию запросить карточку товара через **POST-запрос** в Яндекс Еду.

Это самый спорный этап, потому что:
- открытые базы вроде `barcode-list` и `disai` могут возвращать неточные названия;
- найденное название иногда не совпадает с формулировкой в поиске Яндекс Еды;
- часть товаров вообще не находится.

Но как fallback-механизм этот шаг все равно полезен.

## Анализ состава и Е-добавок

Если данные по товару были успешно найдены, далее они передаются в `ICompositionExplanationProvider`.

На этом этапе приложение:

- анализирует **описание** и **состав** и расчитывает Nutri-Score (полезность продукта), NOVA (оценивает степень и цель промышленной обработки продуктов), и Светофор (данные по составу соли, сахару жиров);
- выделяет **Е-добавки**;
- показывает пояснение по каждой найденной добавке.

### Источник данных по добавкам
База Е-добавок собрана в формате **JSON** и сформирована путем парсинга сайта **proe.info**.

Почему это удобно:
- у добавок есть подробные описания;
- присутствуют синонимы;
- повышается точность распознавания в составе продукта.

На практике это дает хороший результат: приложение почти всегда находит упомянутые в составе добавки и корректно выводит информацию по ним.

## Интерфейс приложения

В приложении есть три основные вкладки:

### 1. Главная
Главная страница выступает агрегатором основных сценариев и показывает:

- недавние сканирования;
- список покупок;
- полезные и новостные данные из Роскачества.

### 2. Сканер
Во вкладке сканера можно:

- отсканировать штрих-код через камеру;
- ввести код вручную;
- открыть карточку продукта с найденной информацией.

### 3. Меню
В меню расположен дополнительный функционал приложения.

#### Список покупок
- добавление товаров вручную;
- голосовой ввод;
- создание напоминаний на выбранные дату и время.

#### Карты лояльности
Поддерживается несколько способов добавления карты:

- ручной ввод;
- сканирование штрих-кода;
- сканирование QR-кода;
- прикрепление фотографии карты.

Если номер карты введен вручную или отсканирован:
- для него генерируется собственный **штрих-код** и **QR-код**.

#### Калькулятор КБЖУ
Позволяет рассчитать дневную норму КБЖУ по пользовательским параметрам.

В основе — формула **Миффлина — Сан Жеора**.

#### Сравнение
В карточке товара доступна кнопка **«Добавить в сравнение»**.

Функционал позволяет:
- собрать группу до **3 продуктов**;
- сравнить их по **КБЖУ**;
- сравнить по количеству **Е-добавок**;
- в будущем — добавить сравнение по **отзывам**;
- показать процентное соотношение по каждой метрике.

#### Настройки
Общие настройки приложения:
- вибрация при сканировании;
- звук уведомлений;
- автосохранение списков;
- и другие пользовательские параметры.

## Технические детали

### Архитектурные идеи
В этом pet-проекте старался закладывать следующие решения:

- использование **нескольких источников данных**;
- fallback-логика для повышения шанса найти товар;
- разделение ответственности через провайдеры для более точного поиска 
- архитектурный подход, удобный для масштабирования;
- возможность подключать новые внешние API без полного пересмотра логики приложения.

Об отзывах: отдельно стоит отметить, что провайдер с отзывами пока не сделан. Во многом это потому, что у меня пока нет четкой картины видения этой системы. В целом, идея в том, чтобы брать отзывы из сайтов Перекрестка, Ленты, Метро, Мегамаркет и т.д, т.е, у крупных магазинов с огромной покупательской базой. Если поотлавливать fetch/XHR у этих сайтов и потом поэкспериментировать и пробрасывать через Postman, то у многих несложно получить общую оценку и в целом отзывы. Основная проблема - это тема из п.5, нет единого наименования товара у открытых баз данных + каждый магазин дает свое относительно уникальное название (добавляет дополнительное описание товара к наименование или тип упаковки). Самые простые по поиску оказались Metro, Spar, Лента и Магнит. Пятерочка и Перекресток требует куки с сессией, которую часто нужно менять. Мегамаркет вообще за 2-3 запроса в минуту уже блокирует, как и Вкусвилл. На крайний случай - парсинг IRecommend и Otzovik.

- Поддержка **DataMatrix**
- Расширение использования **API Роскачества**
- Брать данные из сайта здоровое-питание.рф
- Допилить поиск данных по товару до окончательного вида
- Полноценный модуль **отзывов**, сравнение продуктов не только по КБЖУ и добавкам, но и по пользовательским оценкам
